這是什麼

一個通用的領域知識工作流平台

不論是 PDF、Word、Markdown 還是十幾種程式語言的原始碼，丟進去就好。 系統會聰明地切塊（程式碼用 AST 語法樹、文件用遞迴切分），萃取重點知識，並讓你從三種介面查詢。

🗂️

什麼都能放

文件支援 txt / md / pdf / docx / html / json / csv，以及任何 UTF-8 純文字檔。

🌳

程式碼用 AST 切塊

借鏡 claude-context 的概念，依函式 / 類別 / 方法的語意邊界切塊，而非硬切字數。

🧠

自動萃取領域知識

每個片段都由 Claude 產生摘要與關鍵概念，連同原文一起嵌入，搜尋時更貼近「語意」。

Quick Start

3 分鐘上手

準備好 Python 3.11+ 就能開始。第一次執行時，本地嵌入模型會自動從 HuggingFace 下載一個小模型。

  1. 安裝　建立虛擬環境並安裝套件。 
    # 建立並啟用虛擬環境
python -m venv .venv && source .venv/bin/activate
# 安裝（含開發相依）
pip install -e ".[dev]"
cp .env.example .env   # 依需求調整設定
  2. 放入資料　指向一個檔案或整個資料夾，系統會自動遞迴掃描。 
    opendomainmcp ingest ./your-project
# [load ] ... [split] ... [extract] ... [store]
# Indexed 42 files / 318 chunks.
  3. 開始發問　用一句自然語言搜尋，可加 --kind code 或 --kind text 過濾。 
    opendomainmcp search "檢索是怎麼實作的" --top-k 5
💡 想要純離線、不呼叫 Claude？把環境變數 ODM_EXTRACT_KNOWLEDGE=false 設好即可關閉知識萃取，其餘流程照常運作。
三種使用方式

一套資料，三個介面

CLI、MCP 伺服器、網頁儀表板都跑在同一條管線與同一個資料庫上， 在哪裡放資料、在哪裡查都一致。

⌨️

命令列 CLI

最快的方式。四個指令：ingest、search、stats、clear。

opendomainmcp stats
opendomainmcp clear
🔌

MCP 伺服器

透過 stdio 提供工具給支援 MCP 的客戶端（如 Claude）： ingest_path、search_knowledge、get_stats。

opendomainmcp-server
🖥️

網頁儀表板

所見即所得的主控台，啟動後開瀏覽器即可操作全部功能。

opendomainmcp-web
# http://127.0.0.1:8000
Dashboard Tour

網頁儀表板導覽

五個分頁，涵蓋從觀察、匯入、搜尋到編輯與設定的完整流程。

📊

Dashboard　總覽

一眼看見資料庫現況：已索引的片段數量、使用的嵌入模型與維度、資料存放位置、知識萃取是否開啟。

📥

Ingest　匯入

上傳檔案或指定路徑，匯入過程透過 SSE 即時串流：載入 → 切塊 → 萃取 → 嵌入 → 入庫，每一步都看得到。

🔎

Explore　搜尋探索

輸入問題即得排序後的結果卡片，顯示來源、符號名稱、相似度分數，以及該片段被萃取出的摘要與概念。

✏️

Browse　瀏覽與編輯

表列所有已儲存項目，可逐筆檢視、就地編輯中繼資料（例如修正摘要）或刪除不需要的內容。

⚙️

Settings　設定

直接在網頁調整嵌入後端、萃取模型、切塊大小等參數，變更會寫入 settings.json 並即時生效。

FAQ

常見問題

需要付費的 API 金鑰嗎？

嵌入（向量化）預設使用本地模型，不需金鑰。只有「知識萃取」會呼叫 Anthropic API， 讀取環境變數 ANTHROPIC_API_KEY / ANTHROPIC_BASE_URL；不想用就設 ODM_EXTRACT_KNOWLEDGE=false 關閉。

我的資料存在哪裡？

預設在專案下的 .opendomain/ 目錄：向量資料庫在 chroma/， 網頁可調整的設定覆寫在 settings.json。可用 ODM_DATA_DIR 改路徑。

重複匯入同一份檔案會重複嗎？

不會。每個片段以「來源＋行號＋內容」的雜湊值作為 ID，內容沒變就是同一筆，重新匯入只是覆寫（idempotent）。

遇到不支援或讀不了的檔案會怎樣？

系統採「明確失敗」原則：略過的檔案與發生的錯誤都會被記錄並回報， 而不是默默吞掉，匯入流程會繼續處理其他檔案。

可以換更強的嵌入模型嗎？

可以。嵌入後端是可插拔的，支援本地 (fastembed)、OpenAI、Voyage；在設定頁或環境變數切換即可。 詳見技術文件的「嵌入層」章節。
