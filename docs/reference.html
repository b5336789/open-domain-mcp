Technical Documentation

openDomainMcp 技術文件

一個通用領域知識工作流平台的工程參考：涵蓋架構、資料模型、攝取管線、AST 切塊演算法、 嵌入與向量儲存，以及 REST / MCP / CLI 三種介面的完整規格。

Python ≥ 3.11Chroma (cosine)tree-sitter AST fastembed · 預設 384dFastAPI + SSEMCP (FastMCP)

01系統概觀

openDomainMcp 接收任意資料型態的文件與多種程式語言的原始碼，自動萃取其中的領域知識， 並存入向量資料庫供語意檢索。核心設計原則：

  • 單一事實來源：CLI、MCP 伺服器、Web API 共用同一個執行情境 (Context = 設定 + 向量庫 + 管線)，行為一致。
  • 語意切塊：程式碼依語法樹邊界切塊（借鏡 claude-context），文件以遞迴文字切塊。
  • 知識增強檢索：每個片段由 Claude 產生結構化知識，折入嵌入文字以提升語意命中率。
  • 明確失敗 (Fail Loud)：略過與錯誤一律記錄回報，永不靜默丟棄資料。
  • 可插拔 + 依賴注入：嵌入後端、萃取器、切塊器皆可替換，測試以 fake 元件全離線執行。

02架構與資料流

攝取流程為線性的六階段管線，三個前端介面皆透過 build_context() 取得共用元件：

┌─────────── 介面層 (interchangeable front-ends) ───────────┐ │ cli.py server.py (MCP) api/app.py (FastAPI) │ └───────────────┬──────────────┬───────────────┬────────────┘ └──────────────┴───────────────┘ context.build_context() ┌──────────────┴───────────────┐ │ Context(settings, store, pipeline) └──────────────┬───────────────┘ ▼ load ──▶ split ──▶ extract ──▶ embed ──▶ store ──▶ search loader code/text knowledge embedder Chroma Chroma splitter (Claude) (vector) (upsert) (query)

所有路徑都建立在 opendomainmcp/context.py 的工廠之上，確保不存在第二份管線實作。

原始碼結構

模組職責
context.py組裝並共享 Context（設定 / 向量庫 / 管線）。
config.py以 ODM_ 前綴讀取環境設定，並支援執行期覆寫。
models.py核心資料結構：Chunk、KnowledgeUnit、SearchResult。
ingest/loader.py檔案型別偵測與文字抽取。
ingest/code_splitter.pytree-sitter AST 切塊（含行區間退回）。
ingest/text_splitter.py遞迴文字切塊器。
ingest/pipeline.py攝取編排、目錄走訪、進度事件、錯誤回報。
extract/knowledge.pyClaude 知識萃取與 NullExtractor。
embedding/可插拔嵌入器：local / openai / voyage。
store/chroma_store.pyChroma 封裝：upsert / query / CRUD / stats。
cli.py · server.py · api/app.py三種前端介面。

03資料模型

定義於 models.py，皆為無業務邏輯的 dataclass，於各層之間傳遞。

Chunk

嵌入與儲存的基本單位。kind 為 "code" 或 "text"；程式碼片段攜帶 AST 中繼資料。

欄位型別說明
textstr原始片段內容。
sourcestr來源檔案路徑。
kindstrcode / text。
languagestr?程式語言名稱（文件為 None）。
node_typestr?AST 節點型別，或 block / lines。
symbolstr?定義的符號名稱（函式 / 類別名）。
start_line / end_lineint?1-based 起訖行號。
knowledgeKnowledgeUnit?萃取出的領域知識。

衍生行為：

  • content_hash / id：sha256("{source}:{start}-{end}\n{text}")，作為冪等 upsert 的穩定 ID。
  • embedding_text()：若有知識，回傳「原文 ＋ Summary ＋ Concepts」的增強文字；否則回傳原文。
  • metadata()：扁平化、去除 None 的中繼資料字典（Chroma 友善）。

KnowledgeUnit / SearchResult

class KnowledgeUnit:  summary: str; concepts: list[str]; relations: list[str]
class SearchResult:  id: str; text: str; score: float; metadata: dict

04攝取管線

Pipeline.ingest_path(path, progress) 接受單一檔案或目錄。目錄會遞迴走訪， 並跳過版本控制與相依目錄（.git、node_modules、.venv、 __pycache__、dist、build 等）與隱藏檔。

每檔處理與進度事件

處理過程透過可選的 progress 回呼發出事件，Web 端據此以 SSE 串流即時狀態：

stage意義
load開始載入檔案。
split切塊中（code → AST，text → 遞迴）。
extract逐片段萃取知識，detail 含片段數。
embed / store向量化並 upsert 入庫。
skip不支援、二進位或無內容（記入 report.skipped）。
error非預期錯誤（記入 report.errors，流程續行）。
done / report整體完成；report 含彙總計數。
Fail Loud：UnsupportedFileError 歸入 skipped， 其他例外歸入 errors；知識萃取為「盡力而為」，單一片段失敗只記錄不中斷整體攝取。 回傳 IngestReport(files_indexed, chunks_indexed, skipped[], errors[])。

05AST 程式碼切塊

split_code(source, language, path, max_chars) 借鏡 claude-context： 以 tree-sitter 解析，於語意邊界（函式 / 類別 / 方法）切塊，而非任意字元視窗。 語法 grammar 由各 tree-sitter-* wheel 內建（離線可用），並以解析器快取重用。

演算法

  1. 同層走訪 _chunk_siblings：依序檢視節點。連續的「非定義」節點（import、敘述） 會被合併，直到累積跨度超過 max_chars 才切出一塊。
  2. 定義節點：遇到 _DEFINITION_TYPES（如 function_definition、 class_declaration、impl_item…）先沖出目前群組，再單獨成塊並標註 symbol（取 name 欄位或首個 identifier）。
  3. 過大定義 _split_definition：若定義超過 max_chars， 遞迴進入其 body（block / class_body / declaration_list…）切其成員； 若無可辨識 body 則整塊保留，避免遺失符號。
  4. 退回機制：語言無對應 grammar、解析失敗或結果為空（空檔 / 僅註解）時， 改用 _line_fallback 行區間切塊，node_type="lines"，並記錄警告。

支援語言 → grammar

Python、JavaScript、TypeScript、TSX、Java、Go、Rust、C、C++、C#、Ruby、Bash。 語言名稱對應至 tree_sitter_* 模組與其 language*() 工廠函式 （TS 特殊：language_typescript / language_tsx）。

# 每塊攜帶可追溯定位
Chunk(kind="code", language="python", node_type="function_definition",
      symbol="build_context", start_line=25, end_line=36)

06文字切塊與載入

載入與型別偵測（loader.py）

  • 程式碼副檔名 → LoadedDoc(kind="code", language=...)。
  • .pdf 以 pypdf、.docx 以 python-docx、.html/.htm 以 HTMLParser 抽取（略過 script/style）。
  • 已知文字副檔名（txt/md/json/yaml/csv/xml/css…）直接讀取。
  • 未知副檔名：嘗試 UTF-8 解碼；成功即視為文字，失敗則丟出 UnsupportedFileError（多為二進位）。

遞迴文字切塊（text_splitter.py）

RecursiveTextSplitter(chunk_size, chunk_overlap) 以分隔符優先序 ["\n\n","\n",". "," ",""] 由粗到細切分：選擇文本中存在的最粗分隔符切開， 對仍過大的片段遞迴下探，再將小片段合併為約 chunk_size 的塊，並保留 chunk_overlap 字元的前文重疊。chunk_overlap ≥ chunk_size 會拋出 ValueError。

07知識萃取

萃取器介面為 extract(text, kind, language) -> KnowledgeUnit。 由 get_extractor(settings) 依設定回傳：

  • ClaudeExtractor：呼叫 Anthropic messages.create，預設模型由 ODM_EXTRACTION_MODEL 指定（預設 claude-sonnet-4-6）， max_tokens=600。客戶端自環境讀取 ANTHROPIC_API_KEY / ANTHROPIC_BASE_URL。
  • NullExtractor：當 ODM_EXTRACT_KNOWLEDGE=false 時回傳空知識。

輸出契約

系統提示要求模型「僅」輸出 JSON 物件；_parse 會剝除 Markdown code fence、 擷取首尾大括號之間的 JSON 並正規化欄位：

{
  "summary":   "一到兩句描述此片段的主旨",
  "concepts":  ["1–8 個領域術語或實體"],
  "relations": ["A -> B 形式的關係（可為空）"]
}

若輸出不含 JSON 物件則拋出 ExtractionError。萃取結果寫入中繼資料， 並由 Chunk.embedding_text() 折入嵌入文字以增強檢索語意。

08嵌入層

抽象介面 Embedder：embed(texts) -> list[list[float]] 與 dim 屬性。由 get_embedder(settings) 依後端選擇。

後端類別預設模型 / 維度相依
localLocalEmbedderBAAI/bge-small-en-v1.5 · 384dfastembed（ONNX，免金鑰）
openaiOpenAIEmbeddertext-embedding-3-small · 1536dOPENAI_API_KEY + openai
voyageVoyageEmbeddervoyage-3-lite · 512dVOYAGE_API_KEY + voyageai
本地模型於首次使用時自 HuggingFace 下載；雲端後端的 SDK 採延遲匯入， 缺套件或金鑰時明確報錯而非靜默降級。維度由首批嵌入結果探得並快取。

09向量儲存

ChromaStore 封裝 Chroma；集合不掛載 embedding function（向量由本層自行提供）， 使用 cosine 空間（hnsw:space）。預設持久化於 <data_dir>/chroma；測試可注入 EphemeralClient。

方法行為
upsert(chunks)以 chunk.id（內容雜湊）冪等寫入；嵌入 embedding_text()。
search(query, top_k, where)查詢並回傳 SearchResult，score = 1 − cosine_distance。
get_items / get_item分頁瀏覽 / 單筆取得。
update_metadata / delete_item就地更新中繼資料 / 刪除（不存在回傳 False）。
stats()回傳 collection、count、embedder、dim。
clear()刪除並重建集合。

10REST API

由 api/app.py 的 FastAPI 提供，為共用 Context 的薄包裝； 若 api/static/ 存在則於 / 掛載前端 SPA。預設 127.0.0.1:8000（ODM_WEB_HOST / ODM_WEB_PORT 可調）。

方法路徑說明
GET/api/health健康檢查。
GET/api/stats集合統計 + data_dir + extract_knowledge。
POST/api/searchBody：{query, top_k, kind?}，回傳結果陣列。
POST/api/uploadmultipart 上傳；寫入暫存目錄並回傳其路徑。
GET/api/ingest/stream?path=SSE 串流攝取進度（event 名稱即 stage）。
GET/api/items分頁列出（limit, offset, kind?）。
GET/api/items/{id}取得單筆（不存在 404）。
PATCH/api/items/{id}更新中繼資料 {metadata}。
DELETE/api/items/{id}刪除單筆。
GET/api/settings讀取可編輯設定與環境資訊。
PATCH/api/settings更新 {values} 並重建 Context 使其生效。

11MCP 工具

server.py 以 FastMCP 於 stdio 暴露工具，供支援 MCP 的客戶端（如 Claude）呼叫， 共用同一 Context。執行：opendomainmcp-server。

工具參數回傳
ingest_pathpath: strIngestReport 字典（files/chunks/skipped/errors）。
search_knowledgequery, top_k=5, kind?SearchResult 字典陣列。
get_stats—集合統計。

12CLI

進入點 opendomainmcp（argparse 子指令）：

指令說明
ingest <path>攝取檔案或目錄，將進度印至 stderr。
search <query> [--top-k N] [--kind code|text]語意檢索並列出來源、符號、分數、摘要。
stats列出集合統計。
clear清空集合內所有內容。

13設定參考

所有設定使用 ODM_ 前綴，讀自環境變數與 .env。標示為「可編輯」者亦可由 Web 設定頁調整，覆寫值持久化於 <data_dir>/settings.json 並由 get_settings() 疊加套用。

變數預設可編輯說明
ODM_DATA_DIR.opendomain—資料根目錄（chroma / settings.json）。
ODM_COLLECTION_NAMEdomain_knowledge—Chroma 集合名稱。
ODM_EMBEDDER_BACKENDlocal✓local / openai / voyage。
ODM_EMBEDDER_MODELBAAI/bge-small-en-v1.5✓嵌入模型名稱。
ODM_EXTRACT_KNOWLEDGEtrue✓是否啟用 Claude 知識萃取。
ODM_EXTRACTION_MODELclaude-sonnet-4-6✓萃取所用模型。
ODM_CHUNK_SIZE1200✓文字切塊目標大小（字元）。
ODM_CHUNK_OVERLAP150✓文字切塊重疊量。
ODM_CODE_MAX_CHUNK_CHARS2000✓程式碼塊大小上限（觸發遞迴 / 行退回）。
ODM_WEB_HOST / ODM_WEB_PORT127.0.0.1 / 8000—Web 伺服器位址。
憑證類（ANTHROPIC_*、OPENAI_API_KEY、 VOYAGE_API_KEY）與 DATA_DIR 刻意排除於可編輯欄位之外， 不可由 Web 介面變更（EDITABLE_FIELDS）。

14測試策略

測試完全離線：以確定性的 FakeEmbedder（雜湊 bag-of-words，令詞彙重疊驅動 cosine 相似度）、 FakeExtractor 與 Chroma EphemeralClient 取代真實相依，無需網路或模型下載。

  • 單元：AST 切塊於函式 / 類別邊界、符號與行號正確；文字切塊符合大小與重疊；loader 路由與二進位失敗。
  • 整合：攝取小型 fixture，驗證可檢索且重攝取為冪等（雜湊去重）。
  • API：以 httpx / TestClient 驗證 stats、search、items CRUD 與 ingest SSE 串流。
pytest   # 全綠、無網路、無模型下載

15擴充指南

  • 新增程式語言：於 loader.LANGUAGE_BY_EXT 加副檔名對應， 並於 code_splitter._GRAMMARS 註冊 grammar 模組與工廠函式（並加入該 wheel 相依）。
  • 新增嵌入後端：實作 Embedder 子類別，於 embedding.get_embedder 接上分支。
  • 替換萃取器：提供 extract(text, kind, language) 的物件，注入 Pipeline 即可（依賴注入）。
  • 新增前端：透過 build_context() 取得共用管線 / 向量庫，勿另建第二份管線。
