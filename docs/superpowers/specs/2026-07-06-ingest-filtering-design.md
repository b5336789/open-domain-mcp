## Problem

Ingestion currently admits every supported-extension file under a directory, minus a
hardcoded directory skip list (`ingest/pipeline.py:29`) and hidden files. Test code,
generated code, lock files, and vendored dependencies carry little or no business
meaning, yet they consume LLM extraction tokens and inject noise into retrieval.

## Goal

Deterministic, zero-token filtering that keeps non-business files out of the
pipeline, is configurable at three layers, and reports every skipped file with the
rule that skipped it (Fail Loud). No LLM triage, no content heuristics beyond
generated-marker sniffing (YAGNI â€” revisit only with concrete false-inclusion cases).

## Design

### New module: `ingest/filters.py`

A single `IngestFilter` class owns the decision "should this file be ingested".
Both `Pipeline._walk()` (directory ingest) and single-file ingest call it â€”
one decision point, no per-surface divergence.

### Rule layers (all deterministic; precedence CLI > settings > built-in)

1. **Built-in default excludes** (gitignore-style globs, matched against the path
   relative to the ingest root):
   - Tests: `tests/`, `__tests__/`, `test_*.py`, `*_test.py`, `*.test.ts`,
     `*.test.js`, `*.spec.ts`, `*.spec.js`, `conftest.py`, `fixtures/`,
     `__snapshots__/`
   - Generated: `*.min.js`, `*.min.css`, `*.map`, `*_pb2.py`, `*.pb.go`,
     `*.generated.*`, lock files (`package-lock.json`, `yarn.lock`,
     `pnpm-lock.yaml`, `poetry.lock`, `uv.lock`, `Cargo.lock`, `Pipfile.lock`,
     `composer.lock`, `Gemfile.lock`)
   - Vendored: `vendor/`, `third_party/`, `migrations/`
2. **Generated-marker head sniff:** read only the first ~10 lines; skip the file on
   markers such as `@generated`, `Code generated by`, `DO NOT EDIT`,
   `autogenerated`, `auto-generated`. Case-insensitive. Applies only to files that
   passed layer 1.
3. **User rules:**
   - `ODM_INGEST_EXCLUDE` â€” comma/newline-separated glob list, additive over
     built-ins. Runtime-editable from the web Settings page (added to
     `EDITABLE_FIELDS`).
   - CLI `--exclude GLOB` (repeatable) â€” additive for that run.
   - CLI `--no-default-excludes` â€” disables layer 1 (and layer 2) for that run.

### Fail Loud reporting

Skipped files are never silent. The ingest report gains a `filtered` section:
one entry per skipped file, `{path, rule}` where `rule` names the matching glob or
`generated-marker`. CLI and web progress streams surface the count; the full list is
in the final report. This is distinct from the existing "unsupported/binary skipped"
reporting, which stays as-is.

### `--sync` semantics (behavior change)

An excluded file is treated as *not part of the corpus*. Directory ingest with
`--sync` therefore prunes chunks previously ingested from now-excluded files, the
same way it prunes chunks for deleted files. This is the intended meaning of
filtering; the report lists pruned-because-excluded sources separately so the
change is visible.

## Testing

Offline, no network. Build a temp directory tree mixing business code, tests,
generated files, lock files, and vendored dirs; assert:
- only business files reach the pipeline stages;
- the `filtered` report lists every skipped file with the correct rule;
- precedence: CLI `--exclude` adds over settings; `--no-default-excludes` re-admits
  built-in-excluded files; `ODM_INGEST_EXCLUDE` layers over built-ins;
- head-sniff catches a marker file and reports rule `generated-marker`;
- `--sync` prunes chunks of a file that becomes excluded between runs.

## Out of scope (YAGNI)

- `.odmignore` per-repo file
- LLM triage of business relevance
- Content heuristics (barrel files, getter/setter-only files, comment-ratio)
- File-size caps
